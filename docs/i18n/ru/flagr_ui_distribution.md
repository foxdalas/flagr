# Распределения

**Распределение** делит трафик *внутри* сегмента между вариантами — 50/50, 90/10, 100% на один вариант и так далее. Как только сегмент включает сущность по [охвату](flagr_ui_segments), распределение решает, какой именно вариант она получит.

У каждого сегмента своё распределение, оно показано во вкладке **Конфигурация** флага.

## Полоса распределения

Когда у сегмента есть распределение, цветная полоса показывает разбивку: каждый вариант занимает срез по размеру своего процента, а в легенде указаны ключи вариантов и их проценты. Сегмент без распределения вместо этого показывает **Распределение не настроено** — исправьте это через **«изменить»**. Если сохранённое распределение по какой-то причине не даёт в сумме 100%, полоса показывает **⚠ предупреждение** с фактической суммой.

## Изменение распределения

![Диалог «Изменить распределение» с шаблонами](images/ru/distribution.png)

Нажмите **«изменить»** на распределении сегмента, чтобы открыть редактор:

1. **Отметьте варианты**, которые хотите включить. Отмеченный вариант начинается с 0%, и ему можно задать долю.
2. Задайте долю каждого **ползунком** или в числовом поле.
3. Проценты **должны давать в сумме 100%** — пока это не так, предупреждение показывает текущую сумму, а кнопка **Save** остаётся недоступной.
4. Нажмите **Save**, чтобы применить. Варианты, оставленные на **0%, отбрасываются** из сохранённого распределения — чтобы оставить вариант в разбивке, задайте ему ненулевую долю.

## Шаблоны

Редактор предлагает готовые отправные точки в один клик:

| Шаблон | Что задаёт |
|--------|--------------|
| **Поровну** | Делит 100% поровну между выбранными вариантами (например, 33/33/34). |
| **100% контроль** | Ставит 100% на первый вариант и 0% на остальные. |
| **Канарейка 1/99** | 1% на первый вариант, 99% на второй — крошечный канареечный срез. |
| **Постепенно 10/90** | 10% на первый, 90% на второй — небольшой поэтапный охват. |

Для «Канарейки» и «Постепенно» нужно выбрать минимум два варианта. Если при применении шаблона ни один вариант не отмечен, Flagr сначала сам выбирает их все. После применения шаблона ползунки всё ещё можно донастроить.

## Охват и распределение

Это два разных шага, и их часто путают:

- **Охват %** (на [сегменте](flagr_ui_segments)) отвечает на вопрос *«входит ли эта сущность в сегмент вообще?»* — он фильтрует включение.
- **Распределение** отвечает на вопрос *«какой вариант получит включённая сущность?»* — оно делит включённый трафик.

Так, сегмент со **100% охватом** и **распределением 50/50** даёт вариант всем в сегменте — по половине на каждый. Сегмент с **20% охватом** включает лишь 20% подходящих сущностей; распределение затем делит уже *их*. Полная картина — в разделе [Как работает оценка](flagr_evaluation).

!> Подошедший сегмент **без распределения** не возвращает вариант — Flagr предупреждает об этом (см. [Сегменты и таргетинг](flagr_ui_segments)).
