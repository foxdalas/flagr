# Сценарии использования Flagr

**Фича-флаги, A/B-тестирование и динамическая конфигурация** служат одной цели — показать нужный опыт нужной аудитории,
поэтому в Flagr у них общие строительные блоки. По сути, Flagr сводит их к единому понятию
флага, а интеграция в код для всех трёх выглядит почти одинаково.


## Фича-флаги

Самый частый вид фича-флага — простой переключатель on/off. Чаще всего это аварийные выключатели (kill switch), но иногда у флага есть и целевая аудитория. Вот пример на псевдокоде: для заданной сущности (пользователя, запроса или cookie) Flagr выполняет оценку по настройкам флага.

```
evaluation_result = flagr.post_evaluation( entity )

if (evaluation_result.variant_id == new_feature_on) {
    // здесь делаем что-то новое и классное
} else {
    // здесь оставляем как было
}
```

Типичный фича-флаг настраивается в интерфейсе Flagr примерно так:

```
Варианты
  - on
  - off

Сегмент
  - Условия (зависят от вашей аудитории, например state == "CA")
  - Охват: 100%
  - Распределение
    - on: 100%
    - off: 0%
```

## Эксперименты — A/B-тестирование

Чтобы проводить A/B-эксперименты с несколькими вариантами на выбранной аудитории,
код можно связать с Flagr примерно так:

```
evaluation_result = flagr.post_evaluation( entity )

if (evaluation_result.variant_id == treatment1) {
    // показываем вариант treatment 1
} else if (evaluation_result.variant_id == treatment2) {
    // показываем вариант treatment 2
} else if (evaluation_result.variant_id == treatment3) {
    // показываем вариант treatment 3
} else {
    // показываем контрольный вариант
}
```

Типичный флаг для A/B-теста настраивается в интерфейсе Flagr так:

!> Порядок сегментов важен! Сущность попадёт
в **первый** сегмент, все условия которого выполнены.

```
Варианты
  - control
  - treatment1
  - treatment2
  - treatment3

Сегмент
  - Условия (state == "CA")
  - Охват: 20%
  - Распределение
    - control: 25%
    - treatment1: 25%
    - treatment2: 25%
    - treatment3: 25%
Сегмент
  - Условия (state == "NY" AND age >= 21)
  - Охват: 100%
  - Распределение
    - control: 50%
    - treatment1: 0%
    - treatment2: 25%
    - treatment3: 25%
```

## Динамическая конфигурация

**Вложение варианта** можно использовать и для динамической конфигурации — достаточно передать корректный JSON-объект.

!> До версии [v1.1.3](https://github.com/foxdalas/flagr/releases/tag/1.1.3) во вложении-JSON поддерживались только пары ключ-значение типа **string:string**.

Например, можно на лету менять `color_hex` варианта green:

```
evaluation_result = flagr.post_evaluation( entity )
green_color_hex = evaluation_result.variantAttachment["color_hex"]
```

```
Варианты
  - green
    - вложение: {"color_hex": "#42b983"} ИЛИ {"color_hex": "#008000"}
  - red
    - вложение: {"color_hex": "#ff0000"}

Сегмент
  - Условия: null
  - Охват: 100%
  - Распределение
    - green: 100%
    - red: 0%
```
