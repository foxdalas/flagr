# Варианты

**Варианты** — это возможные значения, которые может вернуть флаг: `on`/`off`, `control`/`treatment`, `green`/`blue`/`pink` и так далее. Оценка всегда возвращает один вариант (или ни одного); ваш код ветвится по ключу варианта.

Раздел «Варианты» находится на вкладке «Конфигурация» флага.

![Раздел «Варианты»](images/ru/variants.png)

## Создание варианта

Введите «Ключ варианта» в поле внизу раздела и нажмите «Создать вариант». Ключ должен быть уникальным в пределах флага и может содержать буквы, цифры, дефисы, слеши, точки и двоеточия (до 63 символов) — при некорректном вводе появляется встроенное сообщение, а кнопка остаётся неактивной, пока ключ не станет корректным.

Если у флага ещё нет вариантов, вы увидите заглушку **No variants defined yet**.

## Изменение или удаление варианта

У каждого варианта показаны его `#id` и редактируемый «Ключ варианта»:

- Измените ключ и нажмите «Сохранить вариант» (активна только при наличии несохранённых правок).
- Иконка корзины удаляет вариант.

!> Нельзя удалить вариант, который всё ещё используется в [распределении](flagr_ui_distribution) сегмента. Flagr заблокирует это и попросит сначала удалить сегмент или изменить его распределение — иначе трафик указывал бы на вариант, которого больше не существует.

## Вложение варианта

Каждый вариант может нести «Вложение варианта» — произвольный JSON-объект, возвращаемый вместе с вариантом. Именно так вы доставляете [динамическую конфигурацию](flagr_use_cases): цвета, тексты, лимиты, параметры функций.

Раскройте «Вложение варианта» под вариантом, чтобы открыть редактор JSON и добавить пары ключ/значение, например:

```json
{ "color_hex": "#42b983", "layout": "modern" }
```

Вложение возвращается в ответе оценки в поле `variantAttachment`. Некорректный JSON отклоняется при сохранении, так что сохранить испорченное вложение не получится.

!> Передаёте типизированное значение через [OpenFeature/OFREP](flagr_ofrep)? Поместите его под ключ `value` (например, `{ "value": 42 }`) — OFREP получает значение флага из `attachment.value`.

Далее: распределите трафик по этим вариантам с помощью [Распределений](flagr_ui_distribution).
