# Редактирование флага

Открытие флага переносит вас в его редактор. Эта страница описывает компоновку редактора, как работает сохранение и собственные настройки флага (ключ, описание, вкл/выкл, заметки, теги, удаление). У вариантов, сегментов, распределений, тестирования и истории есть отдельные руководства.

![Карточка флага в редакторе флага](images/ru/flag-editor.png)

## Компоновка

- **Хлебные крошки** — `Flags / <key>`, чтобы вернуться к списку.
- **Закреплённая шапка** (остаётся видимой при прокрутке) показывает ключ флага, индикатор статуса (зелёная точка — включён, красное кольцо — выключен), значок «Несохранённые изменения», когда есть незавершённые правки, и кнопку «Сохранить все изменения».
- **Вкладки** — «Конфигурация» (всё, что вы редактируете), «Поток оценки» (визуальный [трассировщик](flagr_ui_testing)) и «История» ([журнал изменений](flagr_ui_testing)).
- **Навигация по разделам** — пилюли (Флаг · Варианты · Сегменты · Отладка · Настройки) переносят к каждому разделу и подсвечивают, где вы находитесь, по мере прокрутки.

## Как работает сохранение

Flagr **никогда не сохраняет автоматически** — каждое изменение выполняется явно, поэтому ничего не попадает в продакшен случайно.

- У каждого блока (самого флага, каждого варианта, каждого сегмента, каждого условия) есть **собственная кнопка Save**. Она становится активной (синей) только тогда, когда в этом блоке есть несохранённые правки, и неактивной в остальных случаях.
- Кнопка «Сохранить все изменения» в закреплённой шапке сохраняет *всё* незавершённое за один клик. Она устойчива к сбоям: некорректные условия **пропускаются**, и если отдельное сохранение завершится ошибкой, остальные всё равно пройдут. Сводное уведомление сообщает, что из трёх произошло — всё сохранено, часть элементов пропущена (исправьте и повторите) либо часть сохранений завершилась ошибкой.
- Значок «Несохранённые изменения» появляется всякий раз, когда что-то изменено. Если вы попытаетесь покинуть страницу — или закрыть вкладку — с несохранёнными правками, Flagr сначала попросит подтверждение.
- Горячая клавиша: <kbd>Cmd/Ctrl</kbd>+<kbd>S</kbd> запускает «Сохранить все изменения».

!> Правки вступают в силу только после сохранения. Синяя кнопка Save (или значок «Несохранённые изменения») означает, что что-то ещё не сохранено.

## Предупреждения о конфигурации

Если сегмент настроен так, что это незаметно ломает оценку, в верхней части вкладки «Конфигурация» появляется баннер с предупреждением и ссылкой, которая прокручивает страницу прямо к этому сегменту. Отслеживаются два случая: сегмент с **охватом 0%** и сегмент **без распределения** — в обоих подходящие сущности не получают ни одного варианта. См. [Сегменты и таргетинг](flagr_ui_segments).

## Карточка флага

### Ключ и ID

- **ID флага** (`#42`) отображается рядом с кнопкой копирования.
- «Ключ флага» можно редактировать, и у него есть собственная кнопка копирования — это значение, которое ваш код передаёт как `flagKey` при [оценке](flagr_eval_api). Его изменение переименовывает флаг повсюду.

После редактирования ключа или описания нажмите «Сохранить флаг».

### Описание

Произвольное описание, отображаемое в списке и в хлебных крошках. Предназначено исключительно для людей.

### Включён / Выключен

Переключатель в шапке карточки включает или выключает флаг. **Выключенный флаг не возвращает ни одного варианта** ни одному вызывающему — это главный рубильник (и быстрый аварийный выключатель). Его переключение сохраняется немедленно и показывает уведомление `Флаг включён` / `Флаг выключен`.

### Запись данных

- «Запись данных» — когда включена, оценки этого флага записываются в конвейер метрик (Kafka/Kinesis/Pub-Sub или встроенная [Аналитика](flagr_datar)). По умолчанию выключена, настраивается отдельно для каждого флага.
- «Тип сущности» — необязательная метка (например, `user`, `device`), прикрепляемая к этим записям. Его поведение зависит от конфигурации сервера: по умолчанию это поле со **свободным вводом** и автодополнением из уже используемых типов сущностей (плюс вариант `<null>`); если оператор задал фиксированный список (`FLAGR_UI_POSSIBLE_ENTITY_TYPES`), оно превращается в **выпадающий список**, ограниченный этими значениями. Если оставить его пустым, используется тип сущности, переданный во время оценки, — но учтите, что непустое значение здесь **переопределяет** то, что отправляет вызывающий.

### Заметки флага

Редактор Markdown для документирования флага — плана охвата, владельца, ссылок. Переключатель **изменить / просмотр** переключает между написанием и готовым предпросмотром, панель инструментов охватывает обычное форматирование, а кнопка **?** открывает шпаргалку по Markdown. Под полем расположены счётчик символов и статус *Изменить / Просмотр*; блок заметок полностью скрывается, когда заметки пусты и вы их не редактируете.

Он отображает полный **GitHub-flavored Markdown** — включая таблицы и чекбоксы списков задач (`- [ ]` / `- [x]`) — плюс **формулы KaTeX**: строчные `$…$` и блочные `$$…$$`.

Горячие клавиши при редактировании:

| Горячая клавиша | Действие |
|---|---|
| <kbd>Ctrl</kbd>+<kbd>B</kbd> / <kbd>Ctrl</kbd>+<kbd>I</kbd> | Жирный / курсив |
| <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>S</kbd> | Зачёркнутый |
| <kbd>Ctrl</kbd>+<kbd>E</kbd> / <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>E</kbd> | Строчный код / блок кода |
| <kbd>Ctrl</kbd>+<kbd>1</kbd> / <kbd>2</kbd> / <kbd>3</kbd> | Заголовок уровня 1 / 2 / 3 |
| <kbd>Ctrl</kbd>+<kbd>K</kbd> | Вставить ссылку |

### Теги

Добавляйте теги, чтобы группировать и находить флаги. Введите название и нажмите <kbd>Enter</kbd> (автодополнение предлагает существующие теги); нажмите **×** на чипе, чтобы удалить его. Значения тегов подчиняются тем же правилам, что и ключи вариантов — до 63 символов, буквы/цифры/`-`/`/`/`.`/`:` — и некорректное значение отклоняется. Теги задействованы в [поиске](flagr_ui_flags) и [оценке](flagr_eval_api) по тегам.

## Удаление флага

В разделе «Настройки» кнопка «Удалить флаг» открывает подтверждение, в котором нужно **ввести ключ флага** — защита от случайного удаления. Удаление является *мягким*: флаг скрывается и может быть возвращён из раздела «Удалённые флаги» на странице списка (см. [Управление флагами](flagr_ui_flags)).
