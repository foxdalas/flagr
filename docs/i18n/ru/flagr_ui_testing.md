# Тестирование и история

Перед включением флага (и после) можно точно проверить, как он себя ведёт, и просмотреть каждое изменение, которое он прошёл, — всё это со страницы флага, не задевая живой трафик.

## Консоль отладки

**Консоль отладки** (во вкладке **Конфигурация**) выполняет настоящие оценки флага и показывает сырой результат. Она **только для чтения** — не влияет на продакшен-трафик и ничего не записывает.

![Консоль отладки](images/ru/debug-console.png)

В ней два раздела:

- **Оценка** — оценить флаг для одной сущности. В левой панели запрос (`POST /api/v1/evaluation`), в правой — ответ. Отредактируйте JSON запроса — `entityID`, `entityContext` и т. п. — и нажмите **POST /api/v1/evaluation**. В ответе показаны подошедший `variantKey`, `segmentID`, вложение и журнал отладки.
- **Пакетная оценка** — то же самое, но сразу для множества сущностей (`POST /api/v1/evaluation/batch`).

Пользуйтесь этим, чтобы на конкретном примере ответить на вопрос *«что получит этот пользователь?»*. Схема запроса и ответа описана в руководстве [API оценки](flagr_eval_api).

!> Строковые значения условий в контексте берите в кавычки так же, как в самих условиях — см. [Операторы условий](flagr_operators).

## Поток оценки

Вкладка **Поток оценки** визуализирует весь [путь оценки](flagr_evaluation) и позволяет пошагово проследить путь пробной сущности через него.

![Трассировщик потока оценки](images/ru/eval-flow.png)

1. Введите **ID сущности** и **Контекст сущности (JSON)** — например, `{"country":"DE"}`.
2. Нажмите **«Запустить трассировку»**.

После этого Flagr подсветит точный путь, который прошла сущность:

- Проверка **включён ли флаг** (зелёная, если включён, красная, если выключен).
- Каждый **сегмент** сверху вниз со статусом-меткой:

| Статус | Значение |
|--------|---------|
| **✓ Совпал** | Условия выполнились, и сущность попала в охват — этот сегмент назначил вариант. |
| **Исключён по охвату** | Условия выполнились, но сущность не попала в Охват %. |
| **✗ Не совпал** | Условия не выполнились. |
| **Не достигнут** | Более ранний сегмент уже победил, поэтому этот не проверялся. |
| **Ошибка оценки** | Условие не удалось вычислить (например, некорректное выражение или контекст). Сегмент считается несовпавшим. |

- Конечный узел с итогом, а вверху — **баннер итога**: какой вариант назначен или почему его нет (исключён по охвату / ни один сегмент не подошёл / флаг выключен).

Нажмите **«Очистить»**, чтобы сбросить трассировку. Как и Консоль отладки, это никогда не влияет на живой трафик.

## История

Вкладка **История** — это журнал изменений флага. Каждая запись — это **снимок**, сделанный при изменении флага, сначала самые новые, и показывает:

![Снимок истории со сводкой изменения](images/ru/history.png)

- **Сводку** изменения простыми словами (например, *«Охват (beta): 10% → 50%»*, *«Добавлен вариант „treatment“»*, *«Флаг включён»*).
- **Кто** внёс изменение и **когда** (UTC, с подсказкой относительного времени).
- Переключатель **«Показать технический diff»**, открывающий точный JSON-diff в стиле git для этой ревизии.

История загружается при открытии вкладки. Используйте её, чтобы узнать, кто что изменил, и понять, как флаг пришёл к своему текущему состоянию.

> Хотите **восстановить удалённый флаг**? Это делается в списке флагов, а не здесь — см. [Управление флагами](flagr_ui_flags).
